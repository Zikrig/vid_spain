ТЗ: Telegram-бот — посадочная + автоворонка
прогрева на консультацию
1. Назначение
Бот — посадочная точка для платного
рекламного трафика. Задача: превратить клик из
рекламы в подписчика Telegram-канала и
прогреть его до заявки на бесплатную
консультацию.
Логика посадки: реклама → бот → хук на
бесплатный лид-магнит (дорожная карта /
гайд) → для получения нужно подписаться на
канал → далее прогрев на консультацию.
2. Сценарий пользователя
1.Пользователь переходит из рекламы по
ссылке и запускает бота.
2.Бот сохраняет пользователя и метку
источника (из deep-link).
3.Бот показывает хук: оффер лид-магнита +
условие «чтобы забрать — подпишись на
канал» + кнопка получения.
4.По нажатию кнопки бот проверяет подписку
на канал:

o подписан → выдаёт лид-магнит;
o не подписан → показывает напоминание
подписаться с кнопками «Подписаться» и
«Проверить»; по «Проверить» проверка
повторяется.

5.Независимо от статуса подписки
пользователь попадает в единую цепочку
прогрева на консультацию (отложенные
пуш-сообщения по расписанию).
6.Пуши ведут на запись на бесплатную
консультацию через кнопку-CTA.
3. Проверка подписки
• Бот проверяет членство пользователя в
канале при попытке забрать лид-магнит.
• Требуется, чтобы бот был администратором
канала.
• Проверка подписки используется только как
условие выдачи лид-магнита. Прогрев на
консультацию идёт для всех, кто зашёл в
бота, без деления на подписавшихся и
неподписавшихся.
4. Цепочка прогрева (пуши)

• Серия отложенных сообщений,
отправляемых пользователю по заданному
расписанию (таймингам).
• Каждое сообщение может содержать: текст,
изображение, inline-кнопку со ссылкой
(например, на видео-гайд или на запись на
консультацию).
• Тексты пушей — предоставляет заказчик.
• Цепочка едина для всех пользователей и не
ветвится по статусу подписки.
• Возможность остановки/пропуска пуша при
выполнении целевого действия — по
настройке.
5. Отслеживание подписки/отписки
• Бот фиксирует факт подписки на канал (для
выдачи лид-магнита).
• Опционально: отслеживание отписки для
статистики. При отписке пользователь из
цепочки прогрева не исключается.
6. Запись на консультацию и лиды

• Кнопки-CTA в пушах ведут на запись на
консультацию (внешняя форма или анкета
внутри бота — уточняется).
• К ссылке записи прокидывается метка
источника пользователя.
• Клик по CTA фиксируется как событие;
опционально — уведомление менеджеру о
новом лиде.
7. Админ-панель: управление контентом
Ключевое требование. Заказчик должен
самостоятельно, без разработчика:
• редактировать любой пуш: текст,
изображение, кнопку, ссылку;
• изменять тайминги отправки и порядок
сообщений в цепочке;
• включать и отключать отдельные пуши;
• дублировать цепочку под новую
кампанию/оффер.
8. Метки источника (UTM)

• Бот принимает deep-link вида
t.me/bot?start=<источник> и сохраняет метку
к пользователю.
• Метка используется в аналитике и
прокидывается в ссылку записи на
консультацию.
• Поддержка разных меток для разных
рекламных кампаний.
9. Аналитика
• Хранение статуса каждого пользователя:
источник, подписка, получение лид-магнита,
заявка на консультацию.

• Воронка: старт → подписка → выдача лид-
магнита → клик на консультацию, с разрезом

по источнику рекламы.
• Метрики эффективности по каждому пушу
(доставлено / открыто / клик).
• Экспорт данных по лидам и пользователям.
10. Требования к запуску
• Бот подключён к каналу с правами
администратора.

• Настроены deep-link метки для рекламных
кампаний.
• Загружены тексты и изображения пушей,
заданы тайминги.
• Настроена цель записи на консультацию
(форма или анкета).
